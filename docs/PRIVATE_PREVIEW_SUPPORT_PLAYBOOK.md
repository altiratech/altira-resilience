# Altira Resilience Private Preview Support Playbook

## Purpose

Use this playbook during the curated Altira Resilience private preview so testers have one clear feedback path and Altira has one clear owner for support triage.

## Current Owner

- Preview support owner: `Ryan Jameson`
- Feedback inbox: `contact@altiratech.com`

## Current Feedback Path

Ask testers to send preview issues by email to `contact@altiratech.com`.

For the current curated preview, this email path is the support lane. Do not build a separate in-app ticketing flow yet.

Public request-access submissions should not use the tester issue path. Those now come through `https://altiratech.com/products/resilience/#request-access` and move through `docs/PRIVATE_PREVIEW_REQUEST_ACCESS_HANDOFF.md`.

## Ask Testers To Include

- their workspace email
- the launch or exercise name when relevant
- the page or panel where the issue happened
- whether the issue blocks:
  - sign-in
  - assignment / launch work
  - evidence review / closeout
  - or is general product feedback
- screenshot or copied error text when available

## Triage Rule

- Same-day priority:
  - sign-in failures
  - invite / access blockers
  - launch or evidence actions that a tester cannot complete
- Next business day:
  - UX confusion
  - copy feedback
  - workflow suggestions that do not block the preview

## Boundary

- `Settings` should show the support owner, feedback path, and rollout posture.
- `People`, `Materials`, `Exercises`, and `Evidence` still own the actual operational work.
- Revisit a richer feedback or support workflow only after the curated preview cohort proves the need.
